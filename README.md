# Presentation Feedback (웹+앱)

발표 연습을 할 때, 내가 **얼마나 빨리 말했는지**, **“음… 어… 그…” 같은 추임새를 얼마나 썼는지**, **설명이 잘 이어졌는지**를 자동으로 알려주는 서비스입니다.

교수님께서는 본 프로젝트가 **발표 능력 향상(속도/습관어/구성)이라는 명확한 문제를** 가지고, 사용자가 바로 체감할 수 있는 **정량 지표 + 코칭 피드백**으로 해결한다는 점을 확인하실 수 있습니다.

## 한 줄 소개

- **입력**: 영상/음성 파일 업로드(웹/앱)  
- **처리**: Whisper로 말(자막)을 만들고, 규칙/통계로 분석  
- **출력**: 대시보드(속도·추임새·제안)로 피드백 제공

## 무엇을 해주나요? (중학생도 이해하기)

- **말하기 속도**: 너무 빠른지/느린지 알려줘요. (분당 단어 수)
- **추임새(군더더기 말)**: “음, 어, 그, 저기, 막, 약간…” 같은 말을 몇 번 했는지 세어줘요.
- **제안**: “다음엔 이렇게 해보면 좋아요”를 짧게 추천해줘요.
- **(추가 예정)**: 실시간 발표(라이브) 분석, 발표 구조(도입-본론-결론) 코칭, 기록 저장

## “AI 학습”은 어떻게 하죠? (현실적인 방식)

2~3개월 프로젝트에서 새 모델을 처음부터 학습시키는 건 어렵습니다. 대신, 아래 방식으로 **점점 더 좋아지는 서비스**를 만듭니다.

- **1) 규칙 + 통계 (바로 효과)**  
  추임새를 세고, 말 속도(WPM) 같은 수치를 계산해서 즉시 피드백합니다.

- **2) LLM은 ‘코치 역할’만 (학습 없이도 고품질)**  
  Whisper로 만든 자막을 기반으로, “더 짧고 명확한 문장” 같은 코칭을 생성할 수 있습니다.  
  (이때 채점 기준표(루브릭)를 넣어 피드백 톤을 일정하게 유지합니다.)

- **3) 데이터 수집 루프 (프로젝트 성과 포인트)**  
  사용자 동의 하에 “피드백이 도움이 됐나요?” 같은 평가를 모아, 추후 프롬프트/규칙을 개선합니다.  
  즉, 프로젝트 기간에는 **모델 학습 대신 품질 개선 루프**를 구현하는 것이 목표입니다.

## 현재 구현된 기능 (지금 당장 되는 것)

- **업로드 분석**: 파일 업로드 → Whisper STT → 말 속도/추임새/제안 반환
- **UI**: 토스 스타일(밝은 배경 + 카드 + 빨간 포인트) + 하단 탭(홈/녹화/기록/더보기)

## 구현된 내용 / 구현되지 않은 내용 (상세)

아래 목록은 **현재 레포에 실제로 구현된 기능**과 **아직 구현되지 않은 기능(예정)**을 구분해 정리한 것입니다.

### 구현 완료 ✅

- **클라이언트(웹/앱 공용)**
  - **하단 탭 UI**: 홈 / 녹화·업로드 / 기록 / 더보기
  - **파일 업로드 플로우**: 파일 선택 → 서버 업로드 → 결과 화면 이동
  - **결과 화면 표시**: 말하기 속도(분당 단어 수), 추임새 횟수·비율, 제안, 인식된 글
  - **토스 스타일 테마**: 밝은 배경 + 카드 + 빨간 포인트 컬러

- **서버(API)**
  - **업로드 API**: `POST /api/upload` (multipart `file`) 업로드 처리
  - **Whisper STT 연동(OpenAI API)**: `whisper-1`로 자막 생성(segments 포함)
  - **분석(초기 버전)**: WPM 추정/계산, 추임새 카운트, 비율 계산, 간단 제안 문구 생성
  - **헬스체크**: `GET /api/health`
  - **키 없이 서버 실행 가능**: `OPENAI_API_KEY`는 “업로드로 STT 호출할 때”만 필요

### 일부 구현(데모/플레이스홀더) 🟡

- **분석 기록 탭**
  - 화면은 존재하지만, **실제 기록 저장/목록/상세보기는 아직 없음**
  - “예정된 기능” 목록을 UI로 안내만 제공

- **더보기 탭**
  - 공지/FAQ/의견/약관 메뉴 UI는 있으나, **실제 동작(페이지/링크)은 아직 없음**

### 아직 구현되지 않음 (다음 단계) ⏳

- **실시간 발표(라이브) 분석**
  - 모바일/웹에서 실시간 녹음(마이크) → 스트리밍 분석 → 구간별 피드백

- **발표 구성/문맥 고급 분석**
  - 도입-본론-결론 구조 판단
  - 반복/장황함 탐지
  - 핵심 메시지(키 포인트) 추출

- **LLM 코치 기능**
  - 자막 기반 요약/리라이트(더 짧고 명확하게)
  - 루브릭 기반 점수화(일관된 채점 기준)

- **분석 기록 저장(백엔드)**
  - 로그인/사용자 식별(선택)
  - 결과 DB 저장, 목록/상세 조회 API

- **파일 처리 파이프라인 고도화**
  - 대용량 업로드/비동기 작업 큐(작업 상태: 대기/처리중/완료/실패)
  - 오디오 전처리(볼륨 정규화, 무음 구간(VAD) 정밀 계산 등)

- **보안/운영**
  - 업로드 파일 만료/삭제 정책
  - 레이트리밋, CORS 정책 고정, 로그/모니터링

### 범위 밖(옵션) 🌱

- **Whisper 자체 호스팅(로컬/서버 GPU)**
  - 비용 절감은 가능하지만, 초기 개발/운영 복잡도가 증가

## 실행 방법 (Windows)

### 준비물

- Node.js LTS 설치(권장 20 이상)  
  - 안내 문서: `docs/NODEJS_SETUP.md`
- Whisper API 사용 시 OpenAI API 키  
  - 안내 문서: `docs/OPENAI_API_KEY.md`

### 서버 실행

```powershell
cd C:\PresentationFeedback\Server
npm install
copy .env.example .env
npm run dev
```

### 클라이언트 실행(웹)

```powershell
cd C:\PresentationFeedback\Client
npm install
npx expo start --web
```

## 폴더 구조

- `Client/`: React Native(Expo) — 앱/웹 UI
- `Server/`: Express API — 업로드/Whisper/분석
- `docs/`: 설치 및 키 설정 문서
