# Presentation Feedback (웹+앱)

사용자의 발표 음성을 받아 **말하기 속도(WPM), 군더더기/추임새(“음…”, “어…”, “그…”, “저기…”, “막…”, “약간…”) 사용 빈도, 침묵/간격, 문맥/구성 피드백**을 제공하는 서비스의 기본 골격입니다.

## 목표(MVP)

- **입력**
  - 모바일/웹에서 녹화한 음성/영상 업로드(1차 MVP)
  - 실시간(2차: 스트리밍/라이브 세션)
- **처리**
  - Whisper(또는 Whisper 호환 STT)로 자막/타임스탬프 생성
  - 발화 속도/추임새/침묵/구간별 밀도 분석
  - 발표 구조(도입-본론-결론), 주제 일관성, 반복/장황함 등 텍스트 기반 피드백
- **출력**
  - 점수화(가독한 대시보드) + “다음엔 이렇게 말해보세요” 제안

## “AI 학습”을 현실적으로 하는 방법(새 모델 학습 없이)

2~3개월 프로젝트에서 “새 AI 학습”은 비용/데이터/시간 리스크가 큽니다. 대신 아래 3가지를 조합하면 **학습 느낌(개선되는 품질)**을 낼 수 있습니다.

1) **룰 기반 + 통계(즉시 효과, 저비용)**
- 추임새 사전(한국어) + 형태 변형(“그…”, “그런…”, “그 뭐지…”)을 정규화해서 카운트
- 침묵/말 끊김/속도는 Whisper 타임스탬프와 오디오 에너지(간단 VAD)로 계산
- 발표 길이 대비 추임새 비율, 구간별 WPM 변동, “한 문장 평균 길이” 등 지표 제공

2) **LLM(대형 언어모델)로 ‘코치 역할’만 시키기(학습 없이 고품질)**
- STT 결과를 입력으로
  - 요약(핵심 3줄)
  - 논리 구조 피드백(도입/문제/해결/근거/마무리)
  - 중복/장황 문장 리라이트(“더 짧게/더 명확하게”)
- 이때 LLM은 **채점 루브릭(기준표)**과 예시(좋은/나쁜 답변)를 프롬프트에 넣어 일관성을 확보합니다.

3) **미세튜닝/선호학습 “대신” 데이터 수집 파이프라인(프로젝트 성과 포인트)**
- 사용자 동의 하에:
  - (발화/피드백/사용자 수정본) 쌍을 저장
  - 어떤 피드백이 “도움됨/별로”였는지 평가 수집
- 이 데이터는 추후 **프롬프트 개선, 룰 보정, 사전(추임새) 확장, 또는 소규모 파인튜닝**의 근거가 됩니다.
- 즉, 프로젝트 기간에는 “모델 학습”이 아니라 **데이터셋 구축과 품질 개선 루프**를 구현하는 것이 현실적입니다.

## 기술 스택(권장)

- **Client**: React Native (Expo)로 iOS/Android/Web 단일 코드베이스
  - 디자인: 토스 느낌(화이트 기반, 넓은 여백, 카드 UI) + **레드 계열 포인트 컬러**
- **Server**: API 서버 + 비동기 잡(Transcribe/Analyze)
  - 업로드 → 작업 큐 → STT → 분석 → 결과 저장
- **STT**: Whisper
  - 초기: API 사용(운영/성능 안정)
  - 이후: 자체 호스팅(비용 최적화) 가능
